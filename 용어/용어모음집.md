* AKS : Azure Kubernetes Service, Azure에서 제공하는 Managed Kubernetes(Container Ochestration) Service, Docker 컨테이너를 자동으로 배치 / 확장 / 업데이트 수행해주는 Azure의 Kubernetes 클러스터
* Kubernetes : Container를 Ochestration 수행하기 위한 플랫폼. 자원 사용에 따라 Auto Scaling 수행함. 결국, Docker는 컨테이너를 포장하기 위한 도구인 것이고, k8s(Kubernetes)는 포장된 컨테이너들을 자동 관리하기 위한 플랫폼
* Azure Functions : Serverless 함수 실행 서비스. 특정 이벤트가 발생했을 때 자동으로 실행되는 코드 조각.
* Trigger : 무엇이 Azure Functions을 실행 시키는가?
* * HTTP Trigger : HTTP 요청 수신시 실행 -> API 기능 만들기
* * Timer Trigger : Scheduling (CRON)
* * Blob Trigger : Blob Storage에 파일 생성 혹은 변경
* * Queue Trigger : Queue Storage 메시지 -> 비동기 작업 처리'
* Serverless : Application이 상시 기동된 것이 아니라, 특정 요청(Trigger)에 따라서 해당 시점에 코드 블록을 호출하여 요청을 처리
* Evnet Grid : Azure 내/외부에서 발생하는 이벤트를 자동으로 다른 서비스에 전달하는 이벤트 라우팅 서비스.
* Biz Talk Server : 기업용 시스템 간 연동 플랫폼, SAP / Oracle / FTP / HTTP / SOAP / 파일 시스템 / 레거시 시스템 등 서로 다른 시스템을 연결하고 변환하고, 라우팅하는 미들웨어
* Shared Access Signature(SAS) : 스토리지에 직접 키를 주지 않고, 특정 권한만 잠시 빌려주는 임시 출입증
* Azure Service Bus : 순서 보장, DLQ 지원, 트랜잭션 등 엔터프라이즈급 기능을 제공하는 고신뢰 메시지 브로커. MSA간 안정적인 통신을 위한 지원

<h1>Azure Storage</h1>

```
1. Blob Storage
 - 파일 저장 : 이미지, 동영상, JSON, 압축파일 등 모든 비정형 데이터 저장 서비스
 - 유형 : Block Blob, Append Blob, Page Blob
 - Block Blob : 일반 파일(이미지, JSON, 로그 등) - 가장 많이 사용
 - Append Blob : 로그 파일
 - Page Blob : Random I/O 필요한 VHD(가상 디스크 용도)

2. Queue Storage
 - 메시지 큐(비동기 처리)

3. Table Storage
 - Key-Value 기반 NoSQL

4. Azure Files
 - SMB 방식의 파일 공유

5. Soft Delete
 5.1) Azure Blob Storage Soft Delete : 사용자가 Blob, Snapshot, Container를 실수로 삭제했을 때, 복구할 수 있도록 Azure가 일정 기간동안 삭제된 데이터를 보존하는 기능
 5.2) 적용 대상 : Blob, Snapshot, Version, Container
 5.3) 복구 가능 기간 : 1 ~ 365일
 5.4) 기본값 : Off
 5.5) 반영 위치 : Storage Account -> Data Protection
 5.6) 기타 : Key Vault와 무관함. Key Vault는 보안용. 'RSA-HSM 키를 Blob Storage에 저장하고, Sofe Delete를 사용한다'는 오답
```
